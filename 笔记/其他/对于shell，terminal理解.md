# 对于shell，terminal理解

## 1. 什么是命令行界面？

命令行界面（英语：Command-line Interface，缩写：CLI）是在图形用户界面得到普及之前使用最为广泛的用户界面，它通常不支持鼠标，用户通过键盘输入指令，计算机接收到指令后，予以执行。

—— 摘自 [Wikipedia](https://zh.wikipedia.org/wiki/命令行界面)

## 2. 终端 —— 人与机器交互的接口

以前其实指的是和电脑交互的设备，终端硬件，现在其实是泛指**终端模拟器**（模拟以前的终端和那些传统的、不兼容图形接口的命令行程序（比如说 GNU 工具集里的大部分命令）交互的软件），

对于那些命令行 (CLI) 程序，终端模拟器会「假装」成一个传统终端设备；而对于现代的图形接口，终端模拟器会「假装」成一个 GUI 程序。一个终端模拟器的标准工作流程是这样的：

1. 捕获你的键盘输入；
2. 将输入发送给命令行程序（程序会认为这是从一个真正的终端设备输入的）；
3. 拿到命令行程序的输出结果（STDOUT 以及 STDERR）；
4. 调用图形接口（比如 X11），将输出结果渲染至显示器。

控制台实际上也是一个终端，但由于一些历史原因，管理员使用的终端叫控制台Console

GNU/Linux 操作系统中，按下 Ctrl + Alt + F1,F2...F6，可切换内核提供的终端（tty是终端的统称）

## 3.Shell —— 提供用户界面的程序
![shell介绍](..\img\对于shell，terminal理解.png)



终端干的活儿是从用户这里接收输入（键盘、鼠标等输入设备），扔给 Shell，然后把 Shell 返回的结果展示给用户（比如通过显示器）。而 Shell 干的活儿是从终端那里拿到用户输入的命令，解析后交给操作系统内核去执行，并把执行结果返回给终端。
